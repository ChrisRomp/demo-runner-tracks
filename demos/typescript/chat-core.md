# Add functionality to an existing project

## Scenario

A developer has been tasked with making an update to a project, specifically to allow new races to be entered on the site. They are new to the project, and unsure of where to start. With Copilot Chat they can quickly discover where updates need to be made, learn about the project, and make the updates necessary.

## Existing infrastructure

This demo will focus on two files:

- [src/lib/data/race-service.ts](../../src/lib/data/race-service.ts), which is used to interact with Prisma.
- [src/lib/data/race-service.test.ts](../../src/lib/data/race-service.test.ts), which has the associated unit tests.

## Narrative

The core part of a developer's role is to take a feature request and add the necessary code to make that a reality. This includes determining where in the project the code needs to be updated, writing the code, and (typically) generating the tests necessary to validate the code. Let's see how this could happen with our project here. We want to start the process of allowing races to be entered into our website. We'll do this by adding the necessary to code to the back-end service. But first, we need to figure out where that is.
